Projeto acadêmico — Engenharia de Software, UniCesumar

Sua presença digital gera dados. O ABOT propõe transformá-los em respostas.

O ABOT é uma proposta de automação que usaria bots para verificar, em diferentes plataformas, se um conteúdo ou anúncio está realmente visível — e organizaria esses dados em relatórios simples, sem jargão técnico.

O problema

Seus números caíram. As plataformas não vão te explicar por quê.

Sem ferramentas, qualquer queda de alcance parece um mistério — e cada plataforma tem seu próprio algoritmo, suas próprias regras e zero transparência.

📉

Alcance caiu sem aviso

O conteúdo está performando pior, mas não há como saber se é o algoritmo, o horário, o formato ou a concorrência.

🧩

Dados demais, clareza nenhuma

As plataformas mostram métricas soltas, em painéis diferentes, exigindo tempo e conhecimento técnico para cruzar tudo.

🚫

Sem provas para reclamar

Há suspeita de que um anúncio está sendo limitado injustamente, mas não há como comprovar isso para abrir um chamado.

01 · Entrada — o que o usuário informa

O fluxo começa com poucos dados: a URL ou identificador do conteúdo (post, anúncio ou perfil), a plataforma de origem e, opcionalmente, termos de busca relacionados (ex.: nome do produto, hashtags). É a partir dessas informações que o sistema decide qual "adaptador" de API acionar.

  • Plataforma: Instagram, TikTok, Facebook/Instagram Ads, YouTube...
  • Identificador: URL do post, ID do anúncio ou @usuário
  • Termos de busca (opcional): para simular como um usuário encontraria o conteúdo
  • Frequência: varredura única ou agendada (ex.: diária)

02 · Orquestração — a fila de varreduras

Um serviço de agendamento (worker) organiza as solicitações em uma fila e as distribui ao longo do tempo, para não sobrecarregar as APIs externas e respeitar limites de requisições (rate limits). Cada item da fila carrega a plataforma de destino, o que define qual adaptador será chamado.

  • Fila de tarefas (ex.: Celery, BullMQ ou similar) com prioridades
  • Controle de limite de requisições por plataforma e por usuário
  • Reprocessamento automático em caso de falha temporária da API externa
  • Registro de logs de cada tentativa, para auditoria do relatório final

03 · Coleta — um adaptador por plataforma

Esta é a parte central da proposta: cada plataforma digital tem sua própria API, com autenticação, endpoints e formatos de resposta diferentes. O ABOT precisaria de um "adaptador" dedicado para cada uma, todos seguindo uma interface comum (mesmas funções de entrada e saída), para que o restante do sistema não precise saber os detalhes de cada API.

  • Meta Graph API (Instagram/Facebook): dados de posts, contas e anúncios
  • TikTok for Developers (Display API / Research API): vídeos e métricas públicas
  • YouTube Data API: vídeos, visualizações e posição em resultados de busca
  • Cada adaptador converte a resposta da API para um formato interno único

04 · Diagnóstico — normalização e relatório

Depois que os adaptadores retornam os dados em um formato comum, um serviço de normalização calcula as métricas do relatório: quantas tentativas foram feitas, em quantas o conteúdo apareceu, qual a taxa de visibilidade e um diagnóstico textual simples. Esse resultado é o que seria exibido no painel do usuário.

  • Cálculo de taxa de visibilidade e variação no tempo
  • Classificação simples: boa visibilidade, alcance limitado ou crítico
  • Armazenamento do histórico para comparação entre períodos
  • Exportação do relatório (ex.: PDF) para uso interno ou para reportar à plataforma
Integrações planejadas

Cada plataforma, uma API diferente

O maior desafio técnico do ABOT não é o relatório em si — é lidar com APIs distintas, cada uma com autenticação e estrutura de dados próprias. Veja como cada integração funcionaria.

Meta Graph API (Instagram)

Para verificar a presença de um post ou perfil, o adaptador usaria a Graph API da Meta, autenticada via token de acesso vinculado a uma conta de negócios do Instagram conectada a uma página do Facebook.

AutenticaçãoOAuth 2.0 + token de longa duração (conta business)
Endpoint basegraph.facebook.com/v19.0
Dados retornadosalcance, impressões, engajamento do post
Limitecota de chamadas por hora por aplicativo
 
// adaptadores/instagram.js
async function buscarPost(postId, token) {
  const url = `https://graph.facebook.com/v19.0/${postId}` +
    `?fields=permalink,like_count,comments_count,` +
    `insights.metric(reach,impressions)` +
    `&access_token=${token}`;

  const resposta = await fetch(url);
  const dados = await resposta.json();

  return {
    encontrado: true,
    alcance: dados.insights?.data[0]?.values[0]?.value,
    engajamento: dados.like_count + dados.comments_count
  };
}

TikTok for Developers

O adaptador do TikTok usaria a API oficial de desenvolvedores para consultar vídeos públicos de uma conta e suas métricas básicas, autenticando via OAuth 2.0 com escopo de leitura de vídeos.

AutenticaçãoOAuth 2.0 (escopo video.list)
Endpoint baseopen.tiktokapis.com/v2
Dados retornadosvisualizações, curtidas, compartilhamentos
Limitecota diária por aplicativo registrado
 
// adaptadores/tiktok.js
async function buscarVideo(videoId, token) {
  const url = 'https://open.tiktokapis.com/v2/video/query/';

  const resposta = await fetch(url, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      filters: { video_ids: [videoId] },
      fields: ['view_count','like_count','share_count']
    })
  });

  const dados = await resposta.json();
  return { encontrado: dados.data.videos.length > 0,
           metricas: dados.data.videos[0] };
}

YouTube Data API v3

Para verificar se um vídeo aparece em uma busca, o adaptador chamaria o endpoint de busca pública da API, comparando o ID do vídeo do cliente com os resultados retornados para os termos informados.

AutenticaçãoAPI key (chave de aplicação)
Endpoint basewww.googleapis.com/youtube/v3
Dados retornadosposição na busca, visualizações, curtidas
Limitecota diária de "unidades" por chave
 
// adaptadores/youtube.js
async function buscarPosicao(termoBusca, videoId, apiKey) {
  const url = `https://www.googleapis.com/youtube/v3/search` +
    `?part=snippet&q=${encodeURIComponent(termoBusca)}` +
    `&maxResults=10&key=${apiKey}`;

  const resposta = await fetch(url);
  const dados = await resposta.json();

  const posicao = dados.items.findIndex(
    item => item.id.videoId === videoId
  );

  return { encontrado: posicao !== -1, posicao: posicao + 1 };
}

Meta Marketing API (anúncios)

Para anúncios, o adaptador consultaria diretamente os insights da campanha vinculada à conta de anúncios do cliente, trazendo dados oficiais de entrega — úteis para comparar com o que o usuário percebe no dia a dia.

AutenticaçãoOAuth 2.0 (conta de anúncios)
Endpoint basegraph.facebook.com/v19.0/act_<id>
Dados retornadosimpressões, alcance, frequência, status de entrega
Limitecota por conta de anúncios
 
// adaptadores/metaAds.js
async function buscarInsightsAnuncio(adId, token) {
  const url = `https://graph.facebook.com/v19.0/${adId}/insights` +
    `?fields=impressions,reach,frequency` +
    `&access_token=${token}`;

  const resposta = await fetch(url);
  const dados = await resposta.json();

  return dados.data[0] ?? { status: 'sem dados ainda' };
}

Google Analytics (GA4)

Quando o "conteúdo" verificado é uma página do site ou loja do cliente, o adaptador consultaria a Data API do GA4 para trazer dados de tráfego e origem dos acessos — complementando o que as redes sociais mostram com o que de fato chega ao site.

AutenticaçãoConta de serviço Google (OAuth 2.0)
Endpoint baseanalyticsdata.googleapis.com/v1beta
Dados retornadossessões, origem do tráfego, páginas mais vistas
Limitecota de requisições por propriedade GA4
 
// adaptadores/googleAnalytics.js
async function buscarTrafego(propertyId, token) {
  const url = `https://analyticsdata.googleapis.com/v1beta/` +
    `properties/${propertyId}:runReport`;

  const resposta = await fetch(url, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      dateRanges: [{ startDate: '7daysAgo', endDate: 'today' }],
      dimensions: [{ name: 'sessionSource' }],
      metrics: [{ name: 'sessions' }]
    })
  });

  const dados = await resposta.json();
  return dados.rows;
}

Google Search Console

Para conteúdo indexado pelo Google (blog, loja, site), o adaptador consultaria a Search Console API para saber em quais buscas a página aparece, em qual posição média e com qual taxa de cliques — dados que o próprio Google fornece sobre visibilidade orgânica.

AutenticaçãoOAuth 2.0 (propriedade verificada)
Endpoint basesearchconsole.googleapis.com/webmasters/v3
Dados retornadosimpressões, posição média, CTR, consultas associadas
Limitecota diária por propriedade
 
// adaptadores/searchConsole.js
async function buscarDesempenho(siteUrl, pagina, token) {
  const url = `https://searchconsole.googleapis.com/webmasters/v3/` +
    `sites/${encodeURIComponent(siteUrl)}/searchAnalytics/query`;

  const resposta = await fetch(url, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      startDate: '2026-06-01', endDate: '2026-06-13', // exemplo ilustrativo
      dimensions: ['query','page'],
      dimensionFilterGroups: [{ filters: [
        { dimension: 'page', expression: pagina }
      ]}]
    })
  });

  const dados = await resposta.json();
  return dados.rows ?? [];
}

Capacidades adicionais

Além das APIs: o que mantém o diagnóstico atualizado

Estas três capacidades não dependem de uma única API — elas combinam as integrações anteriores para criar a experiência completa do ABOT.

Bots autônomos navegando em plataformas

Para o que as APIs oficiais não cobrem — como saber se um post aparece numa busca dentro do próprio app, na visão de um usuário comum — o ABOT propõe bots de navegação automatizada (ex.: Playwright/Puppeteer), que abrem a plataforma, digitam termos de busca e registram o que aparece na tela.

TecnologiaNavegador automatizado (Playwright)
Açãoabrir app/site, buscar termo, rolar resultados
Dados retornadosposição do conteúdo nos resultados, se apareceu ou não
Cuidadosrespeitar termos de uso e limites de cada plataforma
 
// bots/buscaAutonoma.js
async function buscarComoUsuario(termo, idConteudo) {
  const navegador = await playwright.chromium.launch();
  const pagina = await navegador.newPage();

  await pagina.goto('https://www.instagram.com/explore/search/');
  await pagina.fill('input[type="search"]', termo);
  await pagina.waitForSelector('.resultados');

  const resultados = await pagina.$$eval(
    '.resultados a', els => els.map(e => e.href)
  );

  await navegador.close();
  return { encontrado: resultados.some(r => r.includes(idConteudo)),
           posicao: resultados.indexOf(idConteudo) + 1 };
}

Monitoramento contínuo

Em vez de uma verificação única, o usuário poderia agendar varreduras recorrentes (diárias ou semanais). Um job programado repetiria a coleta para o mesmo conteúdo ao longo do tempo, armazenando o histórico para identificar tendências e quedas.

Tecnologiatarefas agendadas (cron / scheduler)
Frequênciadiária, semanal ou personalizada
Armazenamentohistórico de métricas por data, em banco de dados
Saídagráfico de evolução da visibilidade ao longo do tempo
 
// monitoramento/agendador.js
const cron = require('node-cron');

// roda todos os dias às 06:00
cron.schedule('0 6 * * *', async () => {
  const alvos = await buscarAlvosAtivos();

  for (const alvo of alvos) {
    const resultado = await executarVarredura(alvo);
    await salvarHistorico(alvo.id, resultado);
  }
});

Detecção automática de problemas algorítmicos

Com o histórico de varreduras, o sistema poderia comparar automaticamente o resultado mais recente com a média dos períodos anteriores. Uma queda brusca de visibilidade dispararia um alerta, sugerindo ao usuário um possível problema de alcance.

Entradahistórico de visibilidade por conteúdo
Regraqueda acima de um limite (ex.: 30% vs. média)
Saídaalerta classificado: atenção ou crítico
Usobase para o relatório e para contato com a plataforma
 
// monitoramento/detectarQueda.js
function detectarProblema(historico) {
  const atual = historico[historico.length - 1].visibilidade;
  const anteriores = historico.slice(0, -1);
  const media = anteriores.reduce(
    (s, h) => s + h.visibilidade, 0
  ) / anteriores.length;

  const queda = (media - atual) / media;

  if (queda >= 0.3) return 'critico';
  if (queda >= 0.15) return 'atencao';
  return 'normal';
}

Comparativo

Ferramentas de SEO são ótimas — só não para isto

SEMrush, Ahrefs e Moz são excelentes para quem precisa de SEO e análise de backlinks. O ABOT propõe resolver um outro problema: entender como um conteúdo se comporta dentro das próprias plataformas sociais.

Recurso SEMrush / Ahrefs / Moz ABOT (proposta)
Foco principal SEO, backlinks e busca orgânica Visibilidade real em redes sociais e anúncios
Simula buscas de usuários reais Limitado Sim, via APIs oficiais de cada plataforma
Relatório em linguagem simples Voltado a especialistas Sim, sem jargão técnico
Acessível para pequenos negócios Planos avançados são caros Pensado para esse público desde o início
